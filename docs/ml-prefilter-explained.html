Right now, every single conversation goes to Groq AI for a full audit. That costs quota, takes 8–10 seconds per conversation, and fails when free-tier limits run out. The ML Pre-Filter is a system of 3 checkpoints that sits in front of Groq and handles the easy conversations for free — so Groq only sees the hard ones.

01 · The Problem

Why we need this.

Groq AI is powerful — but it has two real problems when you're running audits every day across 39 agents.

Problem 1 — Rate limits

Free Groq keys have a limit on how many conversations they can process per minute. When 10 agents all run audits at the same time, the keys run out and conversations get skipped.

Problem 2 — Wasted calls

Most conversations are simple and predictable. A "Wrong Number" conversation with 3 messages doesn't need a full 10-second AI analysis — the answer is obvious. Yet we send it to Groq anyway.

The fix

Build 3 cheap, fast, local checkpoints. If any checkpoint is confident enough → score the conversation locally for free. Only send to Groq when none of the checkpoints are confident.

02 · The Full Flow

Before and after the ML filter.

The diagram below shows how a conversation travels through the system — from the SmarterContact inbox, through the 3 ML checkpoints, and either exits with a free local score or escalates to Groq.

The key insight: the 3 checkpoints run locally on your machine in under 20 milliseconds total. They cost nothing. Only conversations that pass all 3 checkpoints without a confident answer ever reach Groq. That means Groq is reserved for the conversations that actually need a full human-like analysis.

03 · The 3 Checkpoints

What each one actually does.

Checkpoint 1 — Phrase Scan  ·  free  ·  <1ms

It reads for specific words.

This is the simplest checkpoint. It scans every message in the conversation looking for exact phrases that have a clear, known meaning — no guesswork needed.

Phrases that flag immediately: "stop texting", "remove me", "unsubscribe", "leave me alone", "don't contact me" — these mean the agent violated compliance. The result is returned instantly: compliance score = 0, red flag raised. Groq never sees this conversation.

Phrases that mean clean exit: Very short conversations where the agent only sent 1–2 messages and there's no response yet — these don't need scoring at all. The system returns a default clean score without any analysis.

Checkpoint 2 — Memory Lookup  ·  free  ·  ~5ms

It compares this conversation to past ones.

Every conversation that has ever been scored by Groq is stored in a memory bank — currently 911 conversations. This checkpoint converts the new conversation into a mathematical "fingerprint" (a list of 384 numbers that represents its meaning), then searches the memory bank for the most similar past conversations.

If the top match is 92% similar or more AND all nearby conversations were "clean" — the system copies the average score from those past conversations and returns it. No Groq needed.

Safety rule: if ANY of the nearby conversations had a red flag, the system refuses to copy the score and escalates to Groq — even if the similarity is very high. It never risks saying "clean" when a similar past conversation was actually flagged.

Checkpoint 3 — Pattern Predictor  ·  free  ·  ~10ms

It predicts the score from learned patterns.

This is a small AI model trained entirely on your own past Groq results. It has learned: "when a conversation looks like X, Groq typically gives it a score of Y." It applies that knowledge to predict all 4 scores without calling Groq.

It only short-circuits if: predicted flag probability is below 15% AND all 4 predicted scores are above 75. If there is any doubt — any sign this conversation might be risky — it passes to Groq.

It retrains automatically as more data comes in — the more conversations that get scored by Groq, the smarter and more confident the predictor becomes.

04 · The Decision Logic

How each checkpoint decides.

For each checkpoint, there are only two possible outcomes: short-circuit (return a free score) or escalate (pass to the next checkpoint or Groq).

06 · Safety Rules

It will never make a dangerous mistake.

The ML system is designed to be conservative. It only short-circuits a conversation when it is highly confident the conversation is clean. It never risks hiding a red flag.

What it CAN do

  • Say "this conversation is clean" and skip Groq
  • Copy scores from very similar past conversations
  • Predict scores based on learned patterns
  • Return an instant result for obvious opt-out violations

What it will NEVER do

  • Say "clean" when any nearby past conversation was flagged
  • Ignore a high flag-risk prediction — always escalates to Groq
  • Override Groq when there is any doubt
  • Short-circuit a conversation it has never seen patterns like before

"The ML can only ever be wrong in one direction: it might send a clean conversation to Groq unnecessarily. It is designed to make that mistake rather than the opposite one — calling a flagged conversation clean."

07 · Rollout Plan

When does it go live?

Right now — Shadow mode

  • ML runs on every conversation and records what it would have decided.
  • Groq still makes the final score — nothing changes for you.
  • System is collecting data to prove it is accurate.
  • 340 conversations stored. Target: 2,000.

Next — Flip it live

  • Run python scripts/eval_prefilter.py to check accuracy.
  • If agreement with Groq is 95%+ → flip PREFILTER_SHADOW_MODE=false in .env.
  • Tier 1 goes live first (lowest risk — exact phrase matching only).
  • Tiers 2 and 3 enabled after verification at 2,000 conversations.

Goal — 50–80% Groq calls skipped

  • Most "Not Interested", "Wrong Number", and simple drip conversations handled locally.
  • Rate limits become a non-issue for normal daily audits.
  • Audit runs faster — ML conversations return in under 1 second.
  • Groq reserved for genuinely complex or risky conversations only.

Long term — Self-improving

  • Dream Worker retrains the predictor model weekly automatically.
  • More conversations = more patterns learned = higher confidence = more savings.
  • Thresholds calibrate themselves based on accuracy data.
  • Eventually handles 80%+ of all conversations with no human intervention.
